language-integrated parser built specifically to parse JSON is usually the best way to process
on-encoded documents. Occasionally, though, there will be a json-encoded field within non-json
ta, e.g. log files in which some entries contain json data in the message field.
en using Rosie to parse such mixed input, it is useful to have a pattern in RPL that will match
json-encoded value so that the structured data in this value can be extracted later, as needed.
VERSION:
PACKAGE: json
IMPORTS: word, num
value
DESCRIPTION:
TESTS:
value accepts "true", "false", "null"
value rejects "ture", "f", "NULL"
value accepts "0", "123", "-1", "1.1001", "1.2e10", "1.2e-10", "+3.3"
value accepts "\"hello\"", "\"this string has \\\"embedded\\\" double quotes\""
value rejects "hello", "\"this string has no \\\"final quote\\\" "
value rejects "--2", "9.1.", "9.1.2", "++2", "2E02."
value accepts "[]", "[1, 2, 3.14, \"V\", 6.02e23, true]", "[1, 2, [7], [[8]]]"
value rejects "[]]", "[", "[[]", "{1, 2}"
value accepts "{\"one\":1}", "{ \"one\" :1}", "{ \"one\" : 1 }"
value accepts "{\"one\":1, \"two\": 2}", "{\"one\":1, \"two\": 2, \"array\":[1,2]}"
value accepts "[{\"v\":1}, {\"v\":2}, {\"v\":3}]"
